# Vega 过程中探讨过的方式

## 编程语言

考虑过 Go、Rust、Node.js、Python。

最终选择 Python：几乎所有环境自带，无需编译，后期 GUI 扩展方便。

性能不是本项目的瓶颈，语言选择不影响检索方案。

## 安装分发

考虑过 `pip install -e .`、`pipx`、手动加 PATH、初始化脚本。

最终选择 `uv tool install`：uv 自动管理隔离环境和 Python，不污染全局，用户只需安装 uv 即可。

## 检索方案

考虑过文件直接扫描、轻量索引、SQLite FTS5。

## 备份方案（等待实现）

`data/` 目录维护独立的 git 仓库用于版本历史和备份，与 Vega 源码仓库完全隔离。

分支策略采用日/月/main 三层结构：

- **日分支**（如 `2026-04-04`）：CLI 每次写入/删除操作自动 commit，频率高但隔离在当天分支
- **月分支**（如 `2026-04`）：CLI 在检测到日期变更时，自动将昨日日分支合并到月分支
- **main**：每月自动或手动合并月分支，保持干净

备份为可选功能。用户配置远程仓库后通过 push 备份，不配置则仅保留本地历史。
