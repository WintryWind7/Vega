以下问题经过评估并做出了有意识的设计选择。如果你发现了新的证据或有新的角度，可以提出，但不要仅基于下面的描述重复报告相同的问题。

1. **edit 的 old/new 精确匹配** — 参考 Claude Code 的 edit 工具设计，要求先 read 再 edit，对自己的改动负责。AI 读取原文后精确匹配不是难事，这种设计简单、可预测、不引入猜测。

2. **JSON stdin 接口** — 所有命令通过 stdin 接收 JSON 参数。这是为 AI 使用设计的——AI 构造 JSON 零成本，且比命令行参数更结构化、更不容易出错。

3. **搜索结果只显示 description** — description 的设计目的就是一条概要，让 AI 判断是否需要 read 正文。这本身就是预览功能。

4. **write 和 edit 分离** — write 只能创建新条目（路径已存在会报错），edit 只能修改已有条目。分离是为了防止误覆盖。
